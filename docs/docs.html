Documentation

Everything you need to get started with Folio and generate application forms in minutes.

💾 Installation

Folio is a portable Windows application — no installation wizard required.

  1. Download Folio.exe from the latest release.
  2. Move the file to a permanent location (e.g. C:\Tools\Folio\).
  3. Double-click Folio.exe to launch.
ℹ️
Windows SmartScreen warning? Click "More info" → "Run anyway". Folio is safe — SmartScreen shows this for any unsigned app.

System requirements: Windows 10 or 11, 64-bit. No internet needed after download.

🔐 Logging in

Folio uses a simple local password to protect the app. On first launch you will be prompted to enter the access password provided by your administrator.

⚠️
Forgot your password? Contact your Folio administrator or the person who set up the app for your agency.

⚡ Quick start

  1. Select Mokwon University from the institution dropdown in the sidebar.
  2. Click ↓ Download Excel Template to get the pre-formatted spreadsheet.
  3. Fill in your student data (one row per student).
  4. Drag and drop the filled Excel file onto the drop zone.
  5. Click Select Photos Folder if students have photos.
  6. Click Generate. A folder picker will appear — choose where to save the DOCX files.
  7. Done. One DOCX per student, ready to print or send.

📥 Download the Excel template

Always use the official template — it has the correct column order that Folio expects.

  1. Open Folio and select your university in the sidebar.
  2. Click the green ↓ Download Excel Template button that appears below the dropdown.
  3. Save the file and open it in Excel or Google Sheets.
ℹ️
Do not rename or reorder the columns. Folio reads by column position, not by header name.

📸 Adding photos

Folio embeds student photos directly into the DOCX file. You have two ways to provide photos:

Option 1 — Name-based (automatic)

Put photo files in a folder. Name each file exactly after the student's full_name value in column B:

photos/
  Mohammed Rahman.jpg
  Fatima Begum.jpg
  Karim Hassan.png

When generating, click Select Photos Folder and point to that folder. Folio will find each student's photo automatically.

Option 2 — Explicit path (column R)

In column R (photo_path), enter the photo path for each student. Can be:

  • Relative: photos/photo1.jpg — relative to the selected photos folder
  • Absolute: C:\Users\HP\Desktop\photos\stu1.jpg
ℹ️
Supported formats: JPG, JPEG, PNG. If no photo is found, the form is still generated with a blank photo cell — no error.

🚀 Generating forms — step by step

  1. Select university — choose from the dropdown in the left sidebar.
  2. Upload Excel — drag your filled .xlsx file onto the drop zone, or click to browse.
  3. Select photos folder (optional) — click "Select Photos Folder" and choose the folder containing student photos.
  4. Review the student count — Folio shows how many valid rows were detected. Invalid rows (missing name or passport) are listed as warnings.
  5. Click Generate — choose an output folder. Folio creates one DOCX per student.
  6. Review warnings — any rows that were skipped (missing required fields) will appear in the log.

📁 Output files

Each generated file is named:

STU001_Mohammed_Rahman.docx
STU002_Fatima_Begum.docx

Files are saved to the folder you chose during generation. They are standard .docx files — open directly in Microsoft Word or Google Docs.

What's pre-filled in each DOCX

  • Page 1 — Full admission form: name (English + Korean row), gender checkbox, dates, passport, address, school, agency, signature fields, date
  • Page 1 — Semester: Summer ☑ is always pre-ticked
  • Page 2 — Consent form: all three consent checkboxes pre-ticked as YES, applicant signature, date
  • Page 3 — Affidavit of Financial Support: sponsor details (name, address, occupation, relation, contact), sponsor signature, date
  • Photo: embedded in the designated photo cell on page 1 (if provided)

🛠️ Common issues

"SmartScreen prevented an unrecognized app from starting"

Click More info → Run anyway. This appears because Folio is an independent app and not yet code-signed with a commercial certificate.

App opens but shows a blank white screen

Try right-clicking Folio.exe → Properties → Unblock → OK, then relaunch.

My Excel file is rejected / 0 students detected

  • Make sure you're using the template downloaded from inside Folio (not a self-made file).
  • Check that row 1 is the header row and data starts from row 2.
  • Column B (full_name) and column G (passport_number) must not be empty.

Photos not appearing in the generated DOCX

  • Make sure the photo file name exactly matches the student's full_name (case-sensitive on some systems).
  • Supported formats: .jpg, .jpeg, .png
  • Alternatively, use column R to provide an explicit file path.

Some fields are blank in the output

Check that the corresponding column in your Excel has a value for that student. Empty cells produce empty fields — Folio does not invent data.
